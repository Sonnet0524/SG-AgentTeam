## 快速参考

### 研究工作流程

```
发现框架设计问题
  ↓
分析问题本质
  ↓
设计解决方案
  ↓
与用户讨论确认
  ↓
编写研究报告
  ↓
提炼方法论
```

### 关键原则

- **纯抽象视角** - 不涉及具体实现
- **框架层面** - 关注设计思路
- **指导价值** - 为框架设计提供思路
- **通用性** - 独立于具体项目
