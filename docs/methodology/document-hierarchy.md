# 文档分层体系

> 📖 Agent Team Framework的核心方法论之一

---

## 概述

文档分层体系是基于Context窗口最小化原则设计的，目标是通过分层披露信息，最小化Agent启动时的context占用，同时保证信息的完整性和可用性。

---

## 三级文档设计

### Level 0 - 必需层（<50行）

**用途**: Agent启动时必须读取

**内容**:
- 当前状态
- 当前任务
- 关键约束
- 下一步动作

**示例**: `agents/*/CATCH_UP.md`

**设计原则**:
- 信息密度高
- 无冗余
- 快速解析

---

### Level 1 - 按需层（<100行）

**用途**: Agent工作时按需读取

**内容**:
- 角色定义
- 核心职责
- 工作规范
- 输出标准

**示例**: `agents/*/ESSENTIALS.md`

**设计原则**:
- 完整但不冗余
- 清晰但有深度
- 实用且可执行

---

### Level 2 - 参考层（不限）

**用途**: Agent遇到问题时参考

**内容**:
- 详细指南
- 最佳实践
- 历史经验
- 案例分析

**示例**: `agents/*/guides/`, `knowledge-base/`

**设计原则**:
- 按需查询
- 索引驱动
- 持续积累

---

## 使用指南

### Agent启动流程

```
1. 读取 Level 0 文档（CATCH_UP.md）
   ↓
2. 了解当前状态和任务
   ↓
3. 开始工作
   ↓
4. 按需读取 Level 1 文档（ESSENTIALS.md）
   ↓
5. 遇到问题时查询 Level 2 文档
```

### 文档维护

**Level 0 文档**:
- 任务开始/结束时更新
- 状态变化时更新
- 每日至少检查一次

**Level 1 文档**:
- 职责变化时更新
- 规范调整时更新
- 定期审查和优化

**Level 2 文档**:
- 完成重要任务后补充
- 发现问题后记录
- 持续积累和优化
